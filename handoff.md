## Known Gaps (explicit, not silently missing)
- Windows agent: security scanning and remediation are real no-ops (log a placeholder message, don't fake findings); only the core telemetry/WS loop and Application Monitoring's Windows path are functional cross-platform (the latter is Linux-only too, actually — see AGENT_SPEC.md).
- No hosted agent binary release/download flow — `install.sh` expects an operator to have already built/placed the `vigilon-agent` binary (via `go build` or a CI artifact) at `/usr/local/bin/vigilon-agent`.
- Compliance v1 covers only the 6 Vigilon-native scanner checks mapped to their nearest CIS control — not a full CIS Linux Benchmark assessment, and no other framework (ISO27001/SOC2/HIPAA/PCI-DSS) is mapped.
- The new Linux-specific scanner checks (root login, world-writable scan, firewalld) and RHEL-family support are verified by `go build`/`go vet` and code review only — not actually executed on a Linux or RHEL-family host this session.
- Teams/Telegram/SMS channels are code-complete and verified to dispatch/fail-gracefully correctly, but none have been proven against a real Teams webhook, Telegram bot, or Twilio account (no credentials available this session).
- WhatsApp alert channel, Network Monitoring (geo map), and Domain Monitoring (DNS/WHOIS/SPF/DKIM/DMARC) remain unimplemented — see `ROADMAP.md`.
- `CostRecommendation`, `ServerInventory` (beyond an empty read endpoint), `RemediationPolicy` remain schema-only — no code writes to them yet.
- No automated tests anywhere in the repo (backend, frontend, or agent) — verification so far is manual (build/typecheck/browser click-through).
- No tracked Prisma migration history (`prisma/migrations/` doesn't exist) — schema changes go through `prisma db push`, which is fine for the current single-environment dev setup but will need `prisma migrate dev` once there's a real production database to protect.

## Ground Rule Going Forward
**Do not mark anything "complete" in this file without having actually run it** (`go build`, `tsc --noEmit`, a manual browser click-through, or an automated test), and say plainly which verification level a claim rests on (ran end-to-end vs. typechecks-only vs. code inspection). This file's previous drift from reality (batches marked done that had no corresponding code) is the reason the restructure sprint had to start with a full re-audit instead of building on top of assumed-working code — don't let it happen again.
